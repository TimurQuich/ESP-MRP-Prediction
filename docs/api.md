# 📚 API Документация

## Обзор

Документация API для приложения прогнозирования межремонтного периода (МРП) электроцентробежных насосов (ЭЦН).

---

## 📦 Основные функции

### `load_data(use_github, uploaded_file)`

**Назначение:** Загрузка данных из GitHub или локального файла.

**Параметры:**
- `use_github` (bool) – True = загрузка с GitHub, False = загрузка локального файла
- `uploaded_file` (UploadedFile) – загруженный файл (если use_github=False)

**Возвращает:** pandas.DataFrame или None

**Пример:**
```python
df = load_data(use_github=True, uploaded_file=None)
```

---

### `preprocess_data(df)`

**Назначение:** Предобработка данных и создание новых признаков.

**Параметры:**
- `df` (pandas.DataFrame) – исходные данные

**Возвращает:**
- `df_encoded` – данные после кодирования категорий
- `df_fe` – данные с новыми признаками

**Создаваемые признаки:**
1. `Qж факт / Qн факт` – соотношение дебитов
2. `Глубина * (% в)` – произведение глубины и обводненности
3. `(% в)^2` – квадрат обводненности
4. `log_Qж` – логарифм дебита жидкости
5. `log_Qн` – логарифм дебита нефти
6. `Глубина * Qж факт` – произведение глубины и дебита

**Пример:**
```python
df_encoded, df_fe = preprocess_data(df)
```

---

### `train_models(X_train, X_test, y_train, y_test)`

**Назначение:** Обучение и сравнение моделей машинного обучения.

**Параметры:**
- `X_train` – обучающие признаки
- `X_test` – тестовые признаки
- `y_train` – целевые значения для обучения
- `y_test` – целевые значения для теста

**Возвращает:**
- `results` – словарь с метриками (MAE, RMSE)
- `trained_models` – словарь с обученными моделями

**Модели:**
1. Linear Regression
2. Random Forest
3. Gradient Boosting
4. Extra Trees

**Пример:**
```python
results, trained_models = train_models(X_train, X_test, y_train, y_test)
```

---

### `optimize_rf_grid(X_train, X_test, y_train, y_test)`

**Назначение:** Оптимизация Random Forest через GridSearchCV.

**Параметры:** те же, что и в `train_models`

**Возвращает:** словарь с результатами оптимизации

**Оптимизируемые параметры:**
- `n_estimators` – количество деревьев (50, 100, 200)
- `max_features` – количество признаков для разбиения ('sqrt', 0.8)
- `min_samples_split` – минимальное количество для разделения (2, 5, 10)
- `min_samples_leaf` – минимальное количество в листе (1, 2, 4)
- `max_depth` – максимальная глубина (None, 10, 20)
- `bootstrap` – использовать бутстрап (True, False)

---

### `create_feature_importance_plot(model, feature_names)`

**Назначение:** Создание графика важности признаков.

**Параметры:**
- `model` – обученная модель (Random Forest)
- `feature_names` – список названий признаков

**Возвращает:** matplotlib figure

---

### `create_scatter_plot(y_true, y_pred, title)`

**Назначение:** График сравнения фактических и предсказанных значений.

**Параметры:**
- `y_true` – фактические значения
- `y_pred` – предсказанные значения
- `title` – заголовок графика

**Возвращает:** matplotlib figure

---

## 📊 Структура данных

**Ожидаемый формат:**

| Колонка | Тип | Описание |
|---------|-----|----------|
| `Модель насоса` | str | Тип установки |
| `Глубина спуска, м` | float | Глубина установки |
| `% в` | float | Обводненность |
| `МРП` | float | Целевая переменная |
| `Qж факт` | float | Дебит жидкости |
| `Qн факт` | float | Дебит нефти |
| `Причина остановки` | str | Причина отказа |

---

## 🔧 Метрики

- **MAE** – средняя абсолютная ошибка
- **RMSE** – корень из среднеквадратичной ошибки
